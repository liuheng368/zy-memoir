## 设计定位

本稿应替换此前泛化的“班级回忆录”方向，改为“钟园幼儿园 2024 届大二班”定制设计。

核心约束：

- **默认强制登录策略**：除管理员路由外，所有入口默认弹出登录浮层；浮层底部支持「跳过登录，仅浏览」进入**游客态**。
- **登录浮层按 URL 区分角色**（v0.3）：`/` → 学生登录浮层；`/teacher` → 老师登录浮层；`/admin?token=` → 管理员直通；**两个浮层完全独立、浮层内不互通**。
- 角色通过地址路由 + 登录态区分：学生（`/`）、老师（`/teacher`）、管理员（`/admin`）、**游客（任一浮层点跳过）**。
- 学生使用"学号 + 姓名"登录，名单来自 `kindergarten_students_en_keys.json`。
- 老师为 3 位，选择姓名登录。
- 管理员页入口在主页右上角，**仅管理员**可见（游客 / 学生 / 老师均不可见）；管理员可管理主页合影。
- **游客态**：可全局浏览（合影、老师录音、学生头像墙、学生 / 老师浮层只读视图），**禁止任何编辑 / 上传 / 删除**；所有 `+ / 换头像 / 编辑 / 长按删除` 入口**统一隐藏**。
- 学生头像墙按学号排序但不显示学号，且不能采用常规格子布局。

   **§4 学生头像墙「星云式」排布规范（v0.2.1 新增）**：

   首版 Figma 走查反馈「星云式」过于模糊，本节补明确示意。

   **示意图**（移动端 375 × ~520 容器，36 个圆形头像散落分布）：

   ```
   ┌────────────────────────────────────────┐
   │ ◯ 同学头像墙                            │
   │                                        │
   │      ●          ●      ●               │
   │   ●     ●           ●        ●         │
   │           ●     ●       ●              │
   │       ●          ●            ●        │
   │   ●        ●         ●     ●           │
   │          ●     ●            ●          │
   │      ●           ●      ●              │
   │   ●      ●            ●        ●       │
   │      ●        ●  ●         ●           │
   │            ●         ●     ●           │
   │       ●         ●        ●     ●       │
   │              ●     ●         ●         │
   │                                        │
   └────────────────────────────────────────┘
   ```

   - **整体形态**：36 个圆形头像在一个**纵向滚动的矩形容器**内"散落"分布，整体**视觉重心居中、向四周松散辐射**（参考首版 Figma `M02` 第 4 区效果）。
   - **不是**：①常规等距 6 × 6 网格；②圆形 / 心形围聚；③螺旋 / 风车（这三种均为 spec **Q-LAYOUT** 备选方案，**v0.1 未采纳**）。
   - **画板尺寸**：`375 × 520`（移动端），`960 × 600`（PC 端，居中摆放）。
   - **头像规格**：
     - **基准直径** `48 px`（移动端）/ `56 px`（PC）；
     - 允许 ±20% 微随机缩放（即 `38 ~ 58 px`），形成大小错落感；
     - **统一**为正圆 + 1 px 描边（白色或浅米色）+ 浅阴影（`0 1px 4px rgba(0,0,0,0.06)`）；
     - 颜色按学生性别（gender）/ 学号 hash 给浅蓝 / 浅粉 / 浅黄等温和配色作为头像底色（已上传头像后被照片覆盖）。
   - **散落算法**（推荐**泊松圆盘采样 / Poisson Disk Sampling**，避免重叠 + 不显得规则）：
     - 最小圆心间距 `≥ 基准直径 × 1.15`（即 `≥ 55 px`，保证不重叠且不挤压）；
     - 容器内边距 `≥ 24 px`，避免头像贴边；
     - **种子固定**（基于学生 id 数组 hash），保证**多次刷新位置一致**，防止家长每次打开看到完全不同的位置。
   - **排序与位置映射**：
     - 学号 1 → 36 顺序作为采样输入序列，**算法决定坐标但学号顺序保留**（用于无障碍 / 屏幕阅读器顺序读出）；
     - **不显示学号文字**；点击头像 → 进入学生浮层。
   - **空间响应式**：
     - 移动端 `375 px` 容器：约 4 ~ 5 列等效密度；
     - PC `960 px` 容器：约 7 ~ 8 列等效密度，头像略大；
     - 容器**高度按学生数动态计算**（36 人下移动端约 `≈ 520 px`），避免出现大片空白。
   - **微动效**（可选）：
     - 入场：错落淡入（每颗头像 `delay = id × 30ms`，整体 ≈ 1.0s 内完成）；
     - 鼠标 hover / 移动端长按 0.3s：当前头像放大 1.1× + 描边变色 + 弹出姓名 tooltip（**非编辑手势**，对游客态也开放）。
   - **空态**：见 spec **Q-DEFAULT-AVATAR**——未上传头像的学生显示默认占位（男 / 女 SVG），**不影响**位置算法。

## 视觉方向

- 色彩：浅米白背景、幼儿园相册暖色、低饱和绿色作为主按钮、浅蓝/粉色区分学生头像。
- 形态：圆形头像、柔和卡片、相册纸张质感，但避免过度卡通。
- 信息密度：移动端优先，直接展示可使用的回忆录主页。

## 严格风险点

1. ~~PRD 同时写了"所有页面必须登录"和"管理员无需登录"，存在表达冲突。~~（**v0.2 已收敛**：除 `/admin` 外默认弹出登录浮层 + 可跳过为游客；管理员仍走 `/admin` 路由直达。）
2. PRD 写"老师 3 位"，但当前仓库只有学生 JSON，没有老师名单数据源。Figma 中只能先用占位老师名。
3. "不同权限根据地址路由进行区分"是很弱的权限边界，**真实实现不能只依赖前端路由**；**v0.2 强化**：游客态的"编辑入口隐藏"仅是 UI 降级，**写接口必须由云函数侧基于 token / 角色二次校验**（详见 plan.md `Q-PLAN-15` + `G14`）。
4. 学生页限制"3 张照片、5 段录音"，管理员是否可覆盖这些限制，PRD 未写清。
5. 图片压缩到 3M 以下、语音 MP3 60 秒属于上传流程关键约束，设计里必须在上传控件附近显式提示。
6. **【新增 v0.2】游客态视觉规范**：
   - "游客"徽标须与「学生 / 老师 / 管理员」彩色徽标在**色相 / 形状**上明确区分（建议中性灰 + 人形剪影 icon），避免被误读为某种登录角色；
   - 「跳过登录，仅浏览」入口须**视觉弱化**（文本按钮、不与"进入"主按钮抢焦点），鼓励用户优先登录；
   - 客态与游客态的浮层 UI 完全一致 —— 设计稿中**不要**为游客态单独绘制"灰显的编辑入口占位"，这会让用户误以为"可以解锁"。
7. **【新增 v0.2】游客态持久化**待定（spec **Q-GUEST-PERSIST** / plan **Q-PLAN-15**）：默认 **不持久化**，每次刷新都重新弹登录浮层；如改为本会话 / 本设备记忆，需在 `M01` 登录浮层增加 **「本设备记住游客态」** 复选框（**默认不画**，待确认后补）。
8. **【新增 v0.2.1】学生头像墙位置稳定性**：散落算法**必须用固定种子**（学生 id 数组 hash），否则每次刷新位置变化会让家长 / 学生**找不到自己**，严重影响 P1 主路径「找到自己 ≤ 3 步」目标。需 plan.md 在 `useStudentWallLayout` composable 中明确该约定。
9. **【新增 v0.2.1】管理员手机端体验**：原 `D02` 桌面稿如直接被压缩到 375 px 视口，3 列网格会变成不可点的拥挤小图，且拖拽上传在 iOS Safari 不可用；必须以 `M06 mobile` 单列大图 + ActionSheet 二次确认 + 系统相册 / 拍照单通道 重做。预估开发任务对应 plan.md `G9` 需拆为 `G9a desktop` + `G9b mobile`。
10. **【新增 v0.3】角色 URL 误打开的兜底完全靠"链接发放策略"，浮层内不再提供任何切换入口**（**v0.3 决议**：浮层完全独立、不互通；详见 spec **Q-LOGIN-CROSS** + plan **Q-PLAN-17**）。
    - 误打开场景：老师收到 `/`（应给学生）/ 学生收到 `/teacher`（应给老师），打开后**只能看到当前 URL 对应的角色浮层**，浮层内**没有**反向跳转入口。
    - **风险**：误打开者可能**误以为只能这一种角色登录**，退而求其次点「↪ 跳过登录」转游客 → 从此失去正确登录身份，主态可编辑相关 AC（AC-2 / AC-3 / AC-6 / AC-8）走不通。
    - **强制兜底（PRD / 运营层）**：
      1. 班级链接发放**严格按角色分发**：家长群发 `/`、老师群发 `/teacher`，**禁止混发**；
      2. 邀请文案 / README 中**明确写出两条 URL**及对应角色；
      3. 顶栏「去登录」气泡文案明确「当前默认为学生入口，老师请直接打开 `/teacher`」；
      4. （可选）`StudentLogin.vue` / `TeacherLogin.vue` 表单上方加一行小字 hint「我是老师，去 `/teacher` 登录」/「我是学生，去 `/` 登录」（**纯文字提示、不带可点击跳转**），保持浮层视觉独立的同时降低误打开成本（plan.md **Q-PLAN-17 方案 C**，待产品决策）。
    - **非强制兜底（设计层）**：浮层内**不允许**出现「我是老师 → / ← 我是学生」可点击链接，否则与 v0.3 决议矛盾，按 AC-18 不通过。
