## 需求描述

为 **钟园幼儿园 2024 届大二班** 制作一个用于沉淀**班级集体回忆**的轻量网站，记录全班同学与老师的**照片、录音**等多媒体素材，长期保存、便于回顾与分享。

**核心动作**：

- **浏览**：任何已登录用户**或游客**进入主页即可看到班级合影轮播、老师风采、全班同学头像墙；点击同学头像可看其照片墙与录音条目（客态 / 游客态展示一致）。
- **编辑（按角色）**：
  - **学生** — 仅维护自己的资料：300 字自我介绍、头像、3 张照片、5 段录音（**主态**）。
  - **老师** — 仅维护自己的资料：头像照片、多段录音。
  - **管理员** — 维护主页合影（增删）。
  - **游客** — **不允许任何编辑 / 上传 / 删除**；所有相关入口**统一隐藏**。
- **回顾**：在聚会、毕业纪念等场景集体观看；移动端 + PC 端均可访问；**未登录的家长 / 亲友亦可游客态浏览**。

**技术栈**（已由 [`prd/prd.md`](./prd/prd.md) 锁定）：

- 前端：**Vue**
- 后端服务：**腾讯云 CloudBase**（云数据库 + 云函数）
- 存储：**CloudBase COS**（图片 + 录音对象存储）

> 详细架构、Vue 版本、组件、Bucket 划分、上传策略、CDN、JSON 名单导入方式等技术决策写入 [`plan.md`](./plan.md) `## 架构设计` / `## 技术方案` / `## 方案澄清事项`；本文件仅做用户可感知层面的描述与产品向澄清。

### 背景与目标

- **场景**：班级日常分享 / 学期末汇报 / 升入小学留念 / 多年后聚会回顾。手机端为主、PC 端兼顾。
- **目标用户**（已确认见 §澄清 Q1）：班级 36 名同学（家长协助）、3 位老师、1 名管理员（家委 / 班主任指定）、**任意拿到分享 URL 的访客（游客态）**。
- **核心目标**：
  - 任意已登录角色能在 **≤3 步** 操作内**找到**目标对象（同学或老师）并**新增 / 删除**一条素材；
  - 主页一眼可见**班级气质**：合影轮播 + 老师风采 + 全班头像墙；
  - 上传后无须刷新即在对应人浮层可见（乐观 UI，**详 Q-OPT**）。
- **非目标（Out of scope，本期）**：
  - 评论 / 点赞 / @人 / 私信等社交互动；
  - 视频上传与播放；
  - 多个班级 / 多租户管理；
  - 一键导出 PDF / 长图回忆录；
  - 复杂权限审计（操作日志、回收站等）。

### 1. 页面结构描述

#### 1.1 主页 `/`（已登录或游客态默认落地页）

页面采用垂直布局，整体结构如下：

```
┌────────────────────────────────────────────────┐
│ ① 顶栏：班级标题  + 当前角色徽标(含游客)  + ⚙   │   ← ⚙ 仅管理员可见；徽标"游客"附「去登录」入口
├────────────────────────────────────────────────┤
│ ② 合影轮播 Banner（多张主页合影 / 自动 + 手动） │   ← 由管理员上传维护
│   ◀  ●○○○                                  ▶  │
├────────────────────────────────────────────────┤
│ ③ 老师风采区                                    │
│   ┌──┐ 老师A           ┌──┐ 老师B  ┌──┐ 老师C  │   ← 头像 + 姓名 + 录音条
│   │👤│ ▶ 录音1 ▶ 录音2 │👤│ ▶ ...  │👤│ ▶ ...  │     录音条点击直接播放
│   └──┘                  └──┘        └──┘        │
├────────────────────────────────────────────────┤
│ ④ 学生头像墙（按 id 排序，不展示学号；不拘泥常规格子）│
│      ◯ ◯  ◯                                     │
│   ◯     ◯    ◯  ◯                               │   ← 散落 / 圆形围聚 / 风车形
│      ◯  ◯ ◯ ◯       …  共 36 个头像              │     具体形态见 Q-LAYOUT
│         ◯ ◯ ◯                                   │
└────────────────────────────────────────────────┘
```

| 编号 | 区域 | 说明 |
| ---- | ---- | ---- |
| ① | 顶栏 | 班级名「钟园幼儿园 2024 届大二班」+ 当前角色徽标（学生 / 老师 / 管理员 / **游客**）；右上角「⚙ 管理员」入口**仅管理员**可见（**Q-ADMIN-ENTRY**）；**游客**徽标点击展开「去登录」入口（**Q-GUEST-LOGIN-ENTRY**）|
| ② | 合影轮播 | 多张合影自动轮播，可手动左右切换；点击是否放大见 **Q-BANNER** |
| ③ | 老师风采 | 3 位老师卡片（头像 + 姓名 + 录音条）；录音条点击**直接播放**（PRD §3.2） |

### 2. 具体区域交互

- **① 顶栏**：
  - 班级标题不可点；
  - 角色徽标点击 → 简短气泡显示「当前以 ◯◯ 身份浏览」+「退出登录」按钮；**游客态**显示「当前为游客身份，仅可浏览」+「**去登录**」按钮（点击重新打开登录浮层 1.2）；
  - **⚙ 管理员入口**：**仅 `/admin` 路由 Cookie / Token 校验通过**时显示（默认对学生 / 老师 / **游客**不可见）；点击进入 1.5。
- **② 合影轮播**：
  - 自动轮播（间隔 **≈3-5s**，**Q-BANNER-INTERVAL**）；
  - 手动左右滑（移动端手势 / PC 鼠标拖拽 / 左右箭头）；
  - 圆点指示器可点击直达；
  - **空态**（管理员尚未上传任何合影）：显示占位图 + 文案「敬请期待班级合影」；
  - 点击图片**是否放大**见 **Q-BANNER-ZOOM**。
- **③ 老师区**：
  - 头像不可点（或点击放大查看，**Q-TEACHER-AVATAR**）；
  - 录音条 = 圆形播放按钮 + 时长 + 简短波形；点击直接播放（同一时间只允许一条在播）；
  - **空态**：老师未上传任何录音时不显示录音条，仅头像 + 姓名。
- **④ 学生头像墙**：
  - 头像可点 → 进入学生浮层（1.3）；
  - **以登录身份判定主 / 客态 / 游客态**：登录学生点**自己**头像 → **主态**；点**其他人** → **客态**；老师 / 管理员点任意学生 → **客态**（**Q-TEACHER-EDIT-OTHER**）；**游客**点任意学生 → **游客态**（与客态展示一致，**所有编辑入口隐藏**）；
  - **空态**：理论上 36 人都在名单中，无空态；新生（名单未导入）情况见 **Q-NEW-STUDENT**；
  - **视觉空态**（学生未上传过头像）：显示默认占位（**Q-DEFAULT-AVATAR**）。
- **登录浮层（1.2）**：
  - **不再有 Tab**；浮层形态由当前 URL 决定（`/` 学生 / `/teacher` 老师）；
  - **`/` 学生浮层**：学号输入框限制数字 1～36；姓名输入框中文最长 8 字；提交时**云函数 `studentLogin`** 在 db `students` 集合校验「学号 + 姓名」（与 [`plan.md` API-2](./plan.md) 一致）；本地真源基线见 [`data/2024_02_student_list.json`](../../../data/2024_02_student_list.json)（同步规则详 [`data/README.md`](../../../data/README.md)）；
  - **`/teacher` 老师浮层**：3 个单选按钮，选定即可「进入」；
  - **浮层互不切换**：两个浮层是两条独立的登录流，**没有**「学生 → 老师」/「老师 → 学生」的浮层内切换控件（**v0.3 决议**，**Q-LOGIN-CROSS**）；
  - **未匹配**：Toast「学号或姓名错误，请核对」并清空输入；
  - **登录态保留**：浏览器 LocalStorage / Cookie，过期策略见 **Q-LOGIN-TTL**；
  - **「跳过登录，仅浏览」**：点击 → 浮层淡出 → 进入主页 **游客态**（`authStore.role='guest'`）；游客态**不写入登录持久化**，刷新后仍会**再次默认弹出登录浮层**（也可勾选「本设备记住游客态」，**Q-GUEST-PERSIST**）。
- **学生浮层（1.3，主态）**：
  - **头像区**：点击「换头像」→ 选图 → 压缩 → 上传 COS → 替换；保留旧文件还是覆盖见 **Q-AVATAR-OVERWRITE**；
  - **自我介绍**：tap 进入编辑态，字数实时统计 / 超 300 字截断或飘红（**Q-INTRO-OVERFLOW**）；blur 自动保存（防抖 1s）+ Toast；
  - **照片**：「+」按钮（≤3 张时可见，已满则灰显）；长按图片出菜单「删除」；点击图片放大预览；
  - **录音**：「+」按钮（≤5 段时可见）→ 进入录音流程（见 §3 录音）；点击波形条试听；长按出「删除」菜单。
- **学生浮层（1.3，客态 / 游客态）**：
  - 所有「编辑 / 删除 / +」入口**隐藏**；长按手势**不响应**；
  - 点击照片放大预览；点击录音条直接播放；
  - 客态与游客态在 UI 上**不做区分**（仅通过顶栏徽标可知当前身份）。
- **教师浮层（1.4，主态）**：
  - 同学生主态的头像 + 录音流程；**无照片墙、无自我介绍**；
  - **录音上限**未明确（PRD「多段」）→ **Q-TEACHER-MAX**。
- **教师浮层（1.4，客态 / 游客态，如开放）**：

### 3. 操作点击后效果

- **登录提交**：成功 → 浮层淡出 → 落到主页；失败 → Toast「学号或姓名错误，请核对」+ 清空学号/姓名（保留 Tab 选项）。
- **跳过登录（游客）**：点击「↪ 跳过登录，仅浏览」→ 浮层淡出 → 落到主页**游客态**；后续任何**编辑入口均不渲染**；若用户强行通过 deeplink 触达带编辑能力的浮层，浮层亦自动降级为客态（**Q-GUEST-DEEPLINK**）。
- **进入学生 / 教师浮层**：从主页**底部弹出（Bottom Sheet 全屏）** 或 **居中放大** 二选一，**Q-OVERLAY-MOTION**；
  - 移动端：默认底部弹出 + 顶部下拉关闭手势；
  - PC 端：居中卡片 + 半透明遮罩；点遮罩或 Esc 关闭。
- **图片上传**：
  - 选定文件 → **客户端压缩到 ≤3 MB**（PRD 强制；详 plan.md `Q-COMPRESS`）→ 进入「上传中」态；
  - **乐观 UI**：浮层照片墙先插入占位缩略图（带「上传中…」遮罩 / 进度环）；
  - 成功 → 占位变正式图，Toast「上传成功」；
  - 失败 → 占位变红色「重试」按钮态；用户可点重试或取消。
- **录音**：
  - 点击「+」录音 → **请求麦克风权限**（首次）→ 进入录音界面（波形动画 + 60s 计时器 + 圆形按钮）；
  - 倒计时到 60s **强制停止**并自动进入预览；
  - 预览态：可试听 / 重录 / 保存；保存路径同照片的「乐观 UI + 失败重试」语义；
  - 上传前**浏览器内编码为 MP3**（PRD 强制；编码方案见 plan.md `Q-MP3-ENCODE`）。
- **录音播放**：
  - 点击波形条直接播放；再次点击暂停；
  - **同一时间只允许一条录音在播**（点击新录音时上一条自动暂停）。
- **删除照片 / 录音**：
  - 长按 → 出确认菜单 → 选「删除」→ 二次确认 → 调云函数删除 COS 文件 + 数据库记录 → 列表项淡出消失；
  - 失败 → 列表项恢复 + Toast「删除失败，请稍后重试」；
  - **游客 / 客态**下长按手势不触发任何菜单。
- **退出登录**：清空本地登录态 → 按当前用户角色对应路由弹回登录浮层（学生回 `/`、老师回 `/teacher`、管理员回 `/admin`）。
- **游客「去登录」**：顶栏徽标气泡点击「去登录」→ **默认跳 `/` 弹学生登录浮层**；如老师需登录，需用户**手动改地址栏**到 `/teacher` 或重新打开老师专用入口链接（浮层内**不提供**学生 ↔ 老师切换）。登录成功后游客态自动结束、升级为对应角色态。
- **网络失败 / 离线**：所有上传操作 fallback 为 Toast「网络异常，请稍后重试」；**离线队列**本期不做（**Q-OFFLINE**）。

### 4. 页面串联关系

- **登录浮层 ↔ 主页**：
  - **未登录**访问 `/` → 弹出 **学生登录浮层**；访问 `/teacher` → 弹出 **老师登录浮层**；访问 `/admin?token=...` → 不弹浮层，走 token 校验；
  - 登录成功（学生 / 老师）→ 浮层关闭，落到 `/`；
  - **跳过登录** → 浮层关闭，落到 `/` 并进入**游客态**；
  - 「退出登录」→ 按角色回到对应 URL 重弹浮层；
  - **游客「去登录」** → 默认跳 `/` 弹学生登录浮层（如需老师登录，由用户手动跳 `/teacher`，浮层内不互切）；
  - **学生浮层 ↔ 老师浮层不互通**：两个浮层是两条**独立**的登录流，浮层内不提供切换入口（**v0.3 决议**，**Q-LOGIN-CROSS**）。
- **主页 ↔ 学生浮层**：
  - 主页学生头像点击 → 浮层弹出（按身份决定主/客态）；
  - 浮层关闭 → 回到主页**原滚动位置**；
  - 浏览器后退键**优先**关闭浮层（拦截 history，**Q-HISTORY**）。
- **主页 ↔ 教师浮层**：
  - 主页老师头像点击 → 浮层弹出（仅自己登录身份匹配的那个老师为主态，其他老师**不可点开浮层**或仅展示客态查看，**Q-TEACHER-OTHER**）；
  - 关闭逻辑同上。
- **主页 ↔ 管理员页**：
  - 仅当走 `/admin` 路由进入并通过校验 → 主页右上角出现 ⚙ 入口；
  - 点击 ⚙ → 全屏跳转 `/admin`；
  - 「← 返回主页」→ 跳回 `/`。
- **录音 / 上传弹层**：
  - 在浮层内部以**模态**叠加；后退键**先**关闭它再关闭浮层。
- **Deeplink**：
  - PRD 未明确分享需求；本期**不**支持外部分享 deeplink 直达学生 / 教师浮层（**Q-DEEPLINK**）。

### 5. 多页面串联

**主路径 P0 — 默认弹出与游客跳过**（**新增**）：

1. 任意非管理员路由打开网站 → **优先弹出登录浮层**；
2. 用户点击底部「↪ 跳过登录，仅浏览」→ 浮层关闭，进入主页**游客态**；
3. 浏览合影、老师录音、点开任意同学头像（客态展示）；**所有编辑入口均不可见**；
4. 顶栏徽标点击「去登录」可随时回到 1.2 登录浮层完成升级。

**主路径 P1 — 学生维护自己**：

1. 浏览器打开 `/` → 自动弹出**学生登录浮层**（亦可跳过为游客，本路径假设学生选择登录）；
2. 输入学号 + 姓名 → 校验通过 → 进入主页；
3. 在④头像墙找到自己（按 id 排序）→ 点击 → **主态**学生浮层弹出；
4. 编辑自我介绍 / 上传头像 / 上传 1 张照片 / 录 1 段 60s 录音；
5. 关闭浮层回主页；任意其他设备/家长再次登录可见。

**主路径 P2 — 学生浏览同学 & 老师**：

1. 学生登录后 → 主页；
2. ② 看合影轮播；③ 点击老师录音条听录音；④ 点击其他同学头像 → **客态**浮层 → 看照片、听录音；
3. 关闭浮层回主页继续浏览。

**主路径 P3 — 老师维护自己**：

1. 浏览器打开 `/teacher` → 自动弹出**老师登录浮层** → 选自己姓名 → 进入主页；
2. ③ 老师区点击**自己**头像 → **教师主态**浮层 → 换头像 / 增删录音；
3. 关闭回主页；同主页可点同学头像看（客态）。

**主路径 P4 — 管理员维护合影**：

1. 管理员通过 `/admin` 路由直达（无需登录浮层；路由 Token 校验，**Q-ADMIN-AUTH**）；
2. 主页右上角出现 ⚙ → 点击进入管理员页；
3. 上传新合影 / 删除旧合影；
4. 「← 返回主页」回 `/` 验证轮播效果。

**分支路径 P5 — 学生误删自己照片**（**新增**：游客态长按无响应）：

1. 主态浮层长按某张照片 → 删除 → 二次确认 → 删除完成；
2. **本期无回收站**（**Q-RECYCLE**）；
3. 用户重新上传即可。

**分支路径 P6 — 重名学生登录冲突**（**待 Q-DUP-NAME**）：

1. `/` 学生浮层输学号 + 姓名 → 名单中**学号唯一**故无冲突；
2. 极端情况 PRD 改名后未同步 JSON → 校验失败 → 走 P1 第 2 步的失败 Toast。

**分支路径 P7 — 角色 URL 走错**（**v0.3 决议：浮层内不互通，下列为兜底走法**）：

1. 老师误打开 `/` → 弹出**学生**登录浮层；浮层内**没有**任何切换到老师浮层的控件；
2. 兜底走法 a：老师**手动**把地址栏改为 `/teacher`（或关闭浮层、重新点击老师专用链接）→ 弹出老师登录浮层；
3. 兜底走法 b（备选）：老师点底部「↪ 跳过登录，仅浏览」→ 进入游客态 → 顶栏「去登录」→ 默认跳 `/`（仍是学生入口）→ 仍需手动跳 `/teacher`；
4. 反向同理：学生误打开 `/teacher` 需手动改回 `/`；
5. **风险**：如老师不知道 `/teacher` 这条 URL 约定，仅看到学生浮层会**误以为只能学生登录**而退而求其次跳过为游客 → 该用户从此失去**老师身份**与编辑能力。**缓解**：分享给老师的链接必须**带 `/teacher`**；详见 figma md 严格风险点 v0.3 与 plan.md **Q-LOGIN-CROSS**。

## 6. 二期功能交互细节（v0.5 · PRD v0.4 新增）

> **范围**：PRD v0.4 「## 二期功能」共 4 条，本节仅描述**用户可感知**的交互；接口、状态字段、文件改动写在 [`plan.md`](./plan.md) 「## 二期实现计划」。
> **不影响一期已有 AC**：所有改动**向后兼容**，已有「长按删除」「轮播 4s」等行为升级为新行为，不留两套并存。

### 6.1 头像上传支持图片裁剪（PRD-2-① · Q-AVATAR-CROP）

- **触发场景**：学生主态在浮层点「换头像 / 上传头像」、老师主态同。
- **流程**：
  1. 选择图片（系统相册 / 相机 / 本地）→ 不再直接进入「压缩 → 上传」流水
  2. **弹出「裁剪」全屏弹层**，背景蒙层显示原图；蒙层中心展示**正方形裁剪框**（学生 / 老师统一方框，CSS 端再渲染圆角；输出比例 1:1）
  3. 用户操作：
     - 移动端：单指拖动平移图片、双指 pinch 缩放
     - PC 端：鼠标拖动平移、滚轮缩放
  4. 底部固定操作条：「取消」「重选」「确认」三按钮；左上角显示「图片裁剪」标题
  5. 「确认」后裁剪框范围导出为 1024×1024 的 JPEG/PNG（按原文件 MIME 决定），交给后续「压缩 → 上传 → 写库」原流水
- **边界**：
  - 裁剪过程**纯本地**，不上传；取消则不消耗任何流量
  - 缩放下限：图片完全填满裁剪框；上限：图片放大到原始像素的 4 倍
  - 平移上限：图片任意一边不得离开裁剪框（防止裁出空白）
  - 极小图（短边 < 100 px）：裁剪框比例不变，缩放下限即为 1×；不阻止上传，但顶部黄色提示条「图片偏小，建议换一张更清晰的照片」
  - 错误：解码失败 → 弹 toast「图片读取失败，请换一张」，回到浮层
- **辅助**：
  - 移动端遵守 `safe-area-inset-bottom`，按钮不被 home indicator 遮挡
  - 弹层关闭走 `overlay-fade` 同款动效，与浮层风格一致

### 6.2 头像点击可放大查看（PRD-2-② · Q-AVATAR-LIGHTBOX）

- **触发场景**：
  - **学生浮层**头像区域：任何模式（主态 / 客态 / 游客态）下点击头像图都可放大
  - **老师浮层**头像区域：同上
  - **不在范围内**：主页学生头像墙的小头像（已绑定「打开浮层」）、卡片上的老师小头像（同）—— 保持原行为
- **行为**：
  - 点击后整屏铺满黑色蒙层（透明度 92%），原图按 `object-fit: contain` 居中展示，最大边长不超过视口
  - **关闭手势**：点击图片以外蒙层任意位置 / 按 ESC / 点击右上角 ✕ 按钮
  - 不影响主态「换头像」按钮：放大蒙层是独立 z-index 顶层
- **边界**：
  - 头像没上传时（默认头像 SVG）也可放大（用户感知一致）
  - 放大期间不阻断录音播放（音频继续，蒙层仅是视觉层）
  - 多次连点不会重复打开（去重：lightbox open 时无视新触发）

### 6.3 每条录音增加显式删除按钮（PRD-2-③ · Q-RECORDING-DELETE-BTN）

- **触发场景**：
  - 学生主态浮层「录音」区每条录音项右侧：增加 **🗑 图标按钮**（owner only）
  - 老师主态浮层同
- **行为**：
  - 短按（< 长按阈值）按钮 → 打开 `ConfirmDialog`「确定删除这段录音？删除后不可恢复。」
  - 与原有「长按 600ms 删除」并存（手机用户习惯保留），二者最终都进 `ConfirmDialog`
  - 客态 / 游客态：删除按钮**完全不渲染**（不是 disabled，是 `v-if` 不出现）
- **可访问性**：
  - 按钮 `aria-label="删除这段录音"`
  - 删除按钮的点击事件 `@click.stop` 阻止冒泡到 `<li>` 的长按计时（防误触发两次确认）

### 6.4 主页合影轮播 5 秒自动翻页（PRD-2-④ · Q-CAROUSEL-AUTOPLAY-5S）

- **触发场景**：主页 [`BannerCarousel`](src/components/home/BannerCarousel.vue) 多张合影时

  - 默认 **5 秒**自动切到下一张（v0.4 之前是 4 秒）
  - 鼠标 hover / 触摸期间暂停（已实现，沿用）
  - 单张时不自动播
  - 用户主动点 ‹ › 或圆点跳转后，自动播计时器重置（已实现，沿用）
